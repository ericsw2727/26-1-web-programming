TODO 조회 및 추가

Supabase의 todo 테이블과 연결하는 실습입니다.

    1-1. TODO 앱에서 저장해야 할 데이터 3가지

    • 할 일 내용(content)
    • 완료 여부(is_done)
    • 생성 날짜(created_at)

    1-2. 브라우저 화면과 데이터 저장소의 역할 차이

    • 브라우저 화면: 사용자가 데이터를 입력하고 결과를 확인하는 역할
    • 데이터 저장소(Supabase): 데이터를 저장하고 다시 불러오는 역할

    1-3. Supabase가 없는 경우 불편한 점

    • 페이지를 새로고침하면 데이터가 사라질 수 있다.
    • 데이터를 장기적으로 저장할 수 없다.
    • 여러 사용자가 같은 데이터를 공유하기 어렵다.

    2-1. id가 왜 필요한지

    • 각 할 일을 구분하기 위해 필요하다.
    • 수정하거나 삭제할 때 어떤 데이터인지 식별할 수 있다.

    2-2. is_done에 들어갈 값 예시

    • true → 할 일을 완료한 상태
    • false → 아직 진행 중인 상태

    2-3. created_at이 없는 경우 불편한 점

    • 언제 작성된 할 일인지 알 수 없다.
    • 오래된 할 일과 최근 할 일을 구분하기 어렵다.
    • 생성 순서대로 정렬하기 힘들어진다.

    3-1. 조회 코드와 추가 코드의 차이

    • 조회 코드는 저장된 데이터를 가져오는 역할을 한다.
    • 추가 코드는 새로운 데이터를 데이터베이스에 저장하는 역할을 한다.

    3-2. .from("todo")의 의미

    • Supabase의 todo 테이블을 선택한다는 의미이다.
    • 어떤 테이블에서 데이터를 조회하거나 추가할지 지정하는 역할을 한다.

    3-3. starter의 TODO를 보고 빠져 있는 기능 예측

    • 기존 할 일 목록 조회 기능
    • 새 할 일 추가 기능
    • 페이지 로드 시 자동으로 목록을 불러오는 기능
